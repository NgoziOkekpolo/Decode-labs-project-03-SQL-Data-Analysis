# SQL Data Analysis – Decode Labs Internship (Project 3)

## Project Overview

This project focuses on analysing an e-commerce dataset using PostgreSQL and pgAdmin 4. The objective was to practise SQL fundamentals 
by retrieving, filtering, sorting and summarising business data to generate meaningful insights.

---

## Tools Used

- PostgreSQL
- pgAdmin 4
- CSV Dataset

---

## Dataset Information

- Rows: 1,200
- Columns: 14

The dataset contains information about customer orders, products, quantities, payment methods, order statuses, referral sources and sales.

---

## Project Workflow

1. Created a PostgreSQL database.
2. Created the **ecommerce** table.
3. Imported the CSV dataset into the table.
4. Converted the **totalprice** column from text to a numeric data type using `ALTER TABLE`.
5. Performed SQL analysis using different SQL commands.

---

## SQL Concepts Used

- SELECT
- WHERE
- ORDER BY
- GROUP BY
- COUNT()
- SUM()
- AVG()
- ALTER TABLE
- REPLACE()

---

## Analysis Performed

- Displayed all records.
- Displayed selected columns.
- Filtered records using different conditions.
- Sorted data.
- Counted total orders.
- Counted orders by status.
- Calculated total sales.
- Calculated sales by payment method.
- Calculated sales by referral source.
- Calculated average order value.
- Calculated average order value by product.
- Calculated average quantity purchased.
- Counted customers by referral source.

---

## Key Findings

- Analysed 1,200 customer orders.
- Calculated total sales of **1,264,761.96**.
- Identified revenue by payment method.
- Compared referral source performance.
- Measured average order values and purchasing behaviour.

---

## Conclusion

This project demonstrates the practical use of SQL for data analysis, from importing and preparing data to generating business insights
through querying, filtering, grouping and aggregation.
